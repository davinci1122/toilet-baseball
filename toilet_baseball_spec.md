# トイレ・ベースボール ⚾ ブラウザアプリ仕様書 v1.0

## 1. コンセプト

トイレの記録（種類 × 滞在時間）を野球の打席に変換するヘルスケア × ゲームアプリ。
「健康にちょうどいい時間」で最高の打撃結果が出る設計により、長居を自然に抑止する。

- 大 ＝ HR狙い（理想 3〜5分）／ 小 ＝ ヒット狙い（理想 〜2分）
- 5分超は「場外ファウル」、10分超は「強制交代」でペナルティ
- 1日1試合、COM（ライバル球団）と塁打数で勝負

## 2. 画面構成

| # | 画面 | 内容 |
|---|------|------|
| 1 | ホーム（計測） | 大 / 小 の巨大2ボタン ＋ 今日のスコア速報 |
| 2 | 計測中 | 経過タイマー ＋ 現在の判定予告（「今止めれば三塁打！」） |
| 3 | 結果演出 | 打球アニメ → 判定スタンプ → 実況テキスト |
| 4 | フィールド | スプレーチャート（日 / 週 / 月切替） |
| 5 | スコアボード | 対COM戦況・月間ペナント表 |
| 6 | 履歴・統計 | 一覧、打率、平均滞在時間、健康インサイト |
| 7 | 設定 | 閾値スライダー、時間帯区分、難易度、サウンド |

ナビゲーション: 下部タブ4つ（ホーム / フィールド / スコア / 設定）。

## 3. 計測フロー

```
[ホーム] --「大」or「小」タップ--> [計測中]
[計測中] --「済んだ！」タップ--> 時間確定 --> 判定 --> [結果演出] --> 保存 --> [ホーム]
```

- 開始時に `startedAt = Date.now()` を保存。経過時間は常に「現在時刻 − startedAt」で算出する
  （setInterval のカウントに頼らないので、画面ロック・バックグラウンドでも正確）。
- 計測中に「キャンセル」ボタン（誤タップ対策、記録は破棄）。
- 30分以上放置 → 復帰時に「押しっぱなしでは？」ダイアログ（時間を手動修正 or 破棄）。
- 押し忘れ用に「あとから手動追加」（種類・時間・時間帯を入力、manual フラグ付き）。
- 時間帯は開始時刻から自動判定: 朝 4:00–10:59 / 昼 11:00–16:59 / 晩 17:00–3:59（設定で変更可）。
- 計測中に10分経過 → バイブ or ローカル通知「そろそろ交代です！」（任意機能）。

## 4. 判定ロジック（デフォルト値・設定で調整可）

### 大（★）

| 滞在時間 | 判定 | 塁打 |
|---|---|---|
| 〜1分 | 単打 | 1 |
| 1〜2分 | 二塁打 | 2 |
| 2〜3分 | 三塁打 | 3 |
| 3〜5分 | ホームラン 🎆 | 4 |
| 5〜10分 | 場外ファウル | 0 |
| 10分〜 | 強制交代（三振扱い）＋健康アラート | 0 |

### 小（・）

| 滞在時間 | 判定 | 塁打 |
|---|---|---|
| 〜2分 | ヒット | 1 |
| 2分〜 | 凡退 | 0 |

- 小にHRは無し（凡例どおり「小＝H、大＝HR」が各種目の最高評価）。
- 閾値は設定画面のスライダーで変更可能（例: HRゾーンを1〜3分に前倒しするストイックモード）。

## 5. フィールド描画仕様（非線形スケール）

場外の面積が大きくならないよう、半径方向を圧縮マッピングする。

```
R1 = 全半径の 58%   // 3分ライン（フェンス）
R2 = 全半径の 88%   // 5分ライン（場外ライン・赤破線）
R3 = 全半径の 100%  // 場外帯（細いグレー帯・目盛りなし）

r(t) =
  t ≤ 3      : (t / 3) × R1
  3 < t ≤ 5  : R1 + ((t − 3) / 2) × (R2 − R1)
  t > 5      : R2 + min((t − 5) / 5, 1) × (R3 − R2)   // 5分超はまとめて薄帯へ
```

- 角度 ＝ 時間帯セクター（右: 朝 / 中央: 昼 / 左: 晩、各30°）。
  同一セクター内は開始時刻の分に応じて角度を分散させ、マークの重なりを防ぐ。
- マーク: 小 ＝ ●（teal）、大 ＝ ★（purple）、10分超 ＝ 赤✕を重ねる。
- マークをタップで詳細ポップ（日時・時間・判定・実況の再表示）。
- 実装はSVG。ズーム不要、扇形1枚に日/週/月の記録を重ね描き。

## 6. 対COM戦仕様

### デイリーマッチ（1日 ＝ 1試合）

- ユーザーが1打席（実トイレ）消化するたび、COMも1打席を自動消化（確率テーブルで抽選）。
- 得点 ＝ 塁打数の合計。日付が変わった時点で試合成立、多い方が勝ち。
- 1日3打席未満はノーゲーム（外出等で記録が薄い日の救済）。

### COM成績テーブル（難易度）

| 難易度 | アウト | H | 2B | 3B | HR |
|---|---|---|---|---|---|
| ルーキー | 55% | 25% | 10% | 3% | 7% |
| プロ | 45% | 27% | 13% | 4% | 11% |
| レジェンド | 38% | 27% | 15% | 5% | 15% |

- COMはマスコットキャラ（例: ライバル球団「ベンピーズ」）。打席結果は1行実況＋ミニアニメで表示。
- 月間ペナント: 勝敗表・勝率・連勝記録。月末に優勝判定 → 表彰演出。

## 7. 演出・実況

- 結果演出: バッターがスイング → 打球が放物線で飛ぶ（着地点 ＝ フィールド上の判定位置）→ 判定スタンプ「ｶｷｰﾝ! HR!!」。
- HR時: 紙吹雪＋花火。効果音はデフォルトOFF（トイレで鳴ると恥ずかしいため、設定でON可）。
- 実況テキストは判定ごとに5パターン程度からランダム。
  例:「朝一番、ライトスタンドへ特大アーチ！」「痛恨の場外…ボールデッドです」「10分経過、ここで無情の交代！」
- 強制交代時は派手にせず、健康Tips（長時間の座位は痔のリスク等）を1行だけ添える。

## 8. データモデル（localStorage / IndexedDB）

```json
{
  "visits": [
    { "id": "v001", "type": "big", "startedAt": 1751600000000,
      "durationSec": 245, "period": "morning",
      "result": "HR", "bases": 4, "manual": false }
  ],
  "games": [
    { "date": "2026-07-04", "userBases": 9, "comBases": 6,
      "outcome": "W", "comLog": ["H", "OUT", "HR", "..."] }
  ],
  "settings": {
    "thresholds": { "big": [1, 2, 3, 5, 10], "small": [2] },
    "periods": { "morning": [4, 11], "noon": [11, 17] },
    "difficulty": "pro", "sound": false
  },
  "records": { "winStreak": 3, "bestDayBases": 12, "monthly": {} }
}
```

- バックエンド不要・完全ローカル。JSONエクスポート / インポートでバックアップ対応。
- ※ Claude Artifacts として実装する場合は localStorage が使えないため、`window.storage` API に置き換える。

## 9. デザインガイド（ポップ × 野球）

- カラー: グラウンド緑 `#2E9E5B` / 土 `#D98E4A` / スタンド amber `#F5A623` / ナイター紺 `#1B2A4A` / アクセント赤 `#E24B4A`
- フォント: M PLUS Rounded 1c（丸ゴシック太字）。数字は電光掲示板風の等幅。
- 計測ボタン: 大 ＝ 茶系 / 小 ＝ 黄系の巨大丸ボタン。直径120px以上、画面下半分に配置（片手・親指操作前提）。
- スコアボードは黒地に黄文字の電光掲示板風。
- マイクロコピーは実況アナ口調で統一。擬音（ｶｷｰﾝ!）・絵文字の多用OK。

## 10. 技術構成

- 単一ページ（React 1ファイル or Vanilla JS）。フィールドはSVG、演出はCSSアニメーション。
- PWA対応: manifest ＋ Service Worker → ホーム画面追加・オフライン動作・アプリ風全画面。
- 外部通信なし（プライバシー面でも安心とアピールできる）。
